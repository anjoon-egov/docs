    <h1>MDMS-Rewritten</h1>
    <div class=3D"Section1">
        <p>Master Data Management Service is a core service that is made av=
ailable on the DIGIT platform. It encapsulates the functionality surroundin=
g Master Data Management. The service creates, updates and fetches Master D=
ata pertaining to different modules. This eliminates the overhead of embedd=
ing the logic of storage and retrieval of Master Data into other modules. T=
he functionality is exposed via REST API</p>
<h2 id=3D"MDMS-Rewritten-Prerequisites">Prerequisites</h2>
<p>Prior Knowledge of Java/J2EE, Spring Boot, advanced knowledge on how to =
operate JSON data would be an added advantage to understand the service.</p=
>
<h2 id=3D"MDMS-Rewritten-Functionality">Functionality</h2>
<p>The MDM service reads the data from a set of JSON files from a pre-speci=
fied location. It can either be an online location (readable JSON files fro=
m online) or offline (JSON files stored in local memory). The JSON files sh=
ould conform to a prescribed format. The data is stored in a map and <stron=
g>tenantID</strong> of the file serves as a key. </p>
<p>Once the data is stored in the map the same can be retrieved by making a=
n API request to the MDM service. Filters can be applied in the request to =
retrieve data based on the existing fields of JSON.</p>
<h2 id=3D"MDMS-Rewritten-SetupandUsage">Setup and Usage</h2>
<p>The spring boot application needs <strong>lombok </strong>extension adde=
d in your IDE to load it. Once the application is up and running API reques=
ts can be posted to the URL.</p>
<h2 id=3D"MDMS-Rewritten-ConfigurationandStructure">Configuration and Struc=
ture</h2>
<p>The config JSON files to be written should follow the listed rules</p>
<ul>
<li><p>The config files should have JSON extension</p></li>
<li><p>The file should mention the tenantId, module name, and the master na=
me first before defining the data</p></li>
</ul>
